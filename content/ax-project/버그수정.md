# 주요 버그 수정 이력

## JS 전체 오동작 (버튼 무반응, "서버 연결 중" 멈춤)

- **원인**: Python 소스 내 JS 정규식 `/^([A-Za-z]:\\)/` 작성 시 `\\`가 렌더링 과정에서 단일 `\`로 변환됨 → 브라우저가 받는 JS는 `/^([A-Za-z]:\)/` (닫히지 않은 캡처 그룹) → 구문 오류로 JS 전체 실행 불가
- **수정**: 정규식 제거, `activeBrand.path[1] === ':'` + `activeBrand.path.slice(0, 3)` 방식으로 드라이브 추출
- **규칙**: Python 문자열 안에 JS 정규식 백슬래시를 절대 넣지 않는다

---

## 이옴 NAS 영상 매칭 불가

- **원인**: 고효율소재 페이지가 메인 `file_index`를 공유 → 마지막 스캔한 브랜드 결과로 덮어써짐
- **수정**: 브랜드별 독립 인덱스 `highperf_nas_idx` 도입 (`build_highperf_nas_idx(brand)` 함수)

---

## 공백 포함 파일명 매칭 실패

- **원인**: 엑셀 소재명의 공백 위치와 NAS 파일명의 공백 위치가 다른 경우
- **수정**: 매칭 전략 2.7 추가 — 양쪽 공백 제거 후 비교

---

## 엑셀 data_start_row 지정 시 헤더 누락

- **원인**: `min_row=338` 등으로 데이터 시작행을 지정하면 4행의 헤더가 스킵됨
- **수정**: `parse_highperf_xlsx`에서 1~(data_start_row-1)행에서 헤더 탐색 후 data_start_row~부터 데이터 읽기로 분리

---

## 앱 시작 시 의도치 않은 자동 스캔

- **원인**: 마지막으로 클릭한 브랜드가 `config.json`에 `enabled:true`로 남아 있어 재시작 시 자동 스캔 실행
- **수정**: 앱 재시작 전 `config.json`의 모든 브랜드를 `enabled:false`로 초기화
