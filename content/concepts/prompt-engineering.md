# 프롬프트 엔지니어링

**프롬프트 엔지니어링**은 LLM에서 원하는 출력을 얻기 위해 입력 텍스트(프롬프트)를 효과적으로 설계하는 기술입니다.

## 핵심 기법

### 1. Zero-shot Prompting
예시 없이 직접 작업을 요청합니다.

```
"이 텍스트의 감정을 긍정/부정/중립으로 분류해줘:
'오늘 날씨가 정말 좋다'"
```

### 2. Few-shot Prompting
몇 가지 예시를 제공해 패턴을 학습시킵니다.

```
감정 분류 예시:
- "너무 행복해" → 긍정
- "정말 화가 난다" → 부정
- "오늘은 평범한 날이야" → 중립

"이 제품은 그냥 그래" → ?
```

### 3. Chain-of-Thought (CoT)
단계별 추론을 유도합니다.

```
"다음 문제를 단계별로 풀어줘:
15명이 각자 3개씩 사탕을 가져왔고,
5명이 각자 2개씩 먹었다면 남은 사탕은?"
```

### 4. System Prompt
모델의 역할과 동작 방식을 정의합니다.

```python
messages=[
    {
        "role": "system",
        "content": "당신은 AI 전문가입니다. 기술적 질문에 정확하고 간결하게 답변하세요."
    },
    {
        "role": "user",
        "content": "트랜스포머 아키텍처를 설명해줘"
    }
]
```

## 좋은 프롬프트 작성 원칙

1. **구체적으로 작성**: 모호한 지시 대신 명확한 요구사항 명시
2. **역할 부여**: "당신은 X 전문가입니다" 형식
3. **출력 형식 지정**: JSON, 리스트, 표 등 원하는 형식 명시
4. **제약 조건 명시**: 길이, 언어, 톤 등
5. **예시 제공**: 원하는 결과의 예시를 보여줌

## 고급 기법

### ReAct (Reason + Act)
추론과 행동을 번갈아 수행합니다.

```
Thought: 질문에 답하려면 현재 날씨 정보가 필요해
Action: 날씨 API 호출 (서울)
Observation: 서울 현재 온도 28°C, 맑음
Thought: 이제 답변할 수 있어
Answer: 서울은 현재 28°C로 맑은 날씨입니다.
```

### Self-Consistency
같은 질문을 여러 번 물어 가장 일관된 답변을 선택합니다.
