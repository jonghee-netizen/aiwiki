# 작업 로그 자동화 안내

이 볼트(`C:\claude\aiwiki`)에서는 Claude Code의 **Stop 훅**을 이용해 `03-Daily/` 작업 로그를 자동으로 남기도록 설정되어 있다.

## 설정 위치

- 훅 정의: `.claude/settings.local.json` (이 컴퓨터 전용 개인 설정, git에는 커밋되지 않음)
- 실행 스크립트: `.claude/hooks/daily-log-check.sh`
- 상태 파일: `.claude/.daily-log-hash` (중복 트리거 방지용, git에는 커밋하지 않음)

## 작동 방식

1. Claude가 응답을 끝내려 할 때마다(Stop 이벤트) 이 스크립트가 먼저 실행된다.
2. `00-MOC`, `01-Notes`, `02-Sources`, `Templates` 폴더에 **아직 커밋되지 않은 변경사항**이 있는지 `git status --porcelain`으로 확인한다.
3. 변경사항이 없으면 → 아무 것도 하지 않고 조용히 종료(정상적으로 응답 마무리).
4. 변경사항이 있고, 그 변경 내용이 **이전에 이미 로그로 남긴 상태와 동일하면**(해시 비교) → 마찬가지로 조용히 종료.
5. 변경사항이 있고 **아직 로그로 남기지 않은 새로운 내용이면** → Claude의 응답 종료를 막고("decision": "block"), "오늘 날짜 `03-Daily/YYYY-MM-DD.md`에 이번 세션에서 만든/수정한 파일을 요약해서 남겨라"라고 지시한다.
6. Claude가 로그를 작성한 뒤 다시 응답을 마치려 하면, 감시 대상 폴더(`03-Daily` 자체는 감시 대상이 아님)에는 더 이상 새로운 변경이 없으므로 훅이 통과되고 정상 종료된다.

## 특징 / 제약

- **매 턴마다 스팸처럼 뜨지 않는다.** 실제로 노트/MOC 파일이 바뀐 세션에서만, 그리고 아직 로그로 안 남긴 변경분에 대해서만 1회성으로 발동한다.
- 감지 기준은 "git status 상 커밋되지 않은 변경사항"이므로, 변경 후 커밋을 해버리면 그 변경분은 더 이상 감지되지 않는다(로그를 남긴 뒤 커밋하는 순서를 권장).
- 이 훅은 **개인 설정(`settings.local.json`)**이라 다른 컴퓨터/다른 사용자에게는 적용되지 않는다. 다른 환경에서도 쓰려면 `.claude/hooks/daily-log-check.sh`는 그대로 두고, `.claude/settings.local.json`의 `hooks.Stop` 블록만 복사해서 해당 환경에 붙여넣으면 된다.
